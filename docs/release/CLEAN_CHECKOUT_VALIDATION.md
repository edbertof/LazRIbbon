# LazRibbon Clean Checkout Validation

This document records the clean-source validation gate for the 2.0 release
path. The goal is to prove that LazRibbon can be extracted into a fresh
directory, audited, built and cleaned without depending on generated files or
the developer's working tree.

## Command

```powershell
powershell -ExecutionPolicy Bypass -File tools/verify_clean_checkout.ps1 -Version 2.0.0 -ReleaseVersion 2.0.0-rc3
```

For a full release-candidate pass, use:

```powershell
powershell -ExecutionPolicy Bypass -File tools/verify_release_candidate.ps1 -Version 2.0.0 -ReleaseVersion 2.0.0-rc3 -OutputDirectory D:\Ribbon4Lazarus
```

## What The Script Does

The clean checkout script performs these steps:

- Release-style source ZIP from the current tree;
- Extract clean source tree into a temporary directory;
- Consistency audit in clean checkout with
  `tools/check_project_consistency.ps1`;
- Register local packages in temporary Lazarus profile so demos resolve
  packages from the extracted tree;
- Full build in clean checkout with the complete package, Skin Editor and demo
  build matrix through
  `tools/build_all_projects.ps1 -CleanArtifacts`;
- Post-build cleanup audit to confirm generated files were removed.

The script accepts `-SkipBuild` for fast audit-only checks and `-KeepWorkspace`
when the temporary clean source tree needs to be inspected after a failure.

## Release Gate

For `2.0.0-rc3`, the clean checkout validation should pass without manual
file copying, local IDE state, globally registered package paths or generated
build artifacts. The source ZIP created by the script is intentionally
temporary; the public release ZIP is still generated by
`tools/verify_release_candidate.ps1`.
