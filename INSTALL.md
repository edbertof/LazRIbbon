# Installing LazRibbon in Lazarus

## Requirements

- Lazarus 4.8 or compatible recent Lazarus version.
- Free Pascal version bundled with your Lazarus installation.
- LCL package installed.

## Clean installation

1. Close Lazarus.
2. Extract this ZIP into a clean directory, for example:

   ```text
   <caminho-do-projeto>\LazRibbon
   ```

3. Open Lazarus.
4. Use **Package > Open Package File (.lpk)**.
5. Open:

   ```text
   packages/LazRibbonRuntime.lpk
   ```

6. Click **Compile**.
7. Open:

   ```text
   packages/LazRibbonDesign.lpk
   ```

8. Click **Compile**.
9. Click **Use > Install**.
10. Rebuild the IDE when Lazarus asks.
11. Restart Lazarus.
12. Confirm that the **LazRibbon** palette appears.

## Reinstalling after replacing files

When testing a new ZIP version:

1. Prefer extracting to a fresh folder.
2. Delete any previous `lib/` output folders if you reused a folder.
3. Compile `LazRibbonRuntime.lpk` first.
4. Compile/install `LazRibbonDesign.lpk` second.

## Object Inspector property map

For new forms, use `docs/quality/COMPONENT_PROPERTY_MATRIX_2_0.md` as the quick
guide to the intended component/property model. Use
`docs/quality/OBJECT_INSPECTOR_PROPERTY_AUDIT_2_0.md` to review which repeated
or inherited property names are intentional, hidden or compatibility-only. These
documents show which component owns each composition link, which properties are
canonical, and which names are kept only for compatibility.

## Build outputs

The package writes compiled units to:

```text
lib/runtime/$(TargetCPU)-$(TargetOS)
lib/design/$(TargetCPU)-$(TargetOS)
```

These folders are generated by Lazarus and are intentionally not shipped with this source distribution.

Before publishing a source ZIP, build and validate it with:

```powershell
powershell -ExecutionPolicy Bypass -File tools/build_all_projects.ps1 -CleanArtifacts
powershell -ExecutionPolicy Bypass -File tools/build_release_zip.ps1
```

You can also audit an existing folder or completed ZIP:

```powershell
powershell -ExecutionPolicy Bypass -File tools/check_release_zip.ps1 <caminho-do-projeto>\LazRibbon.zip
```

## Common checks

If the IDE fails to rebuild:

- check whether another experimental LazRibbon copy is still in the Lazarus package path;
- check whether the old LazToolbar package is installed and causing naming conflicts;
- compile the runtime package before the design-time package;
- delete `lib/` and rebuild;
- send the complete Lazarus compile log when reporting an error.

## Supported environment

Use Lazarus 4.8 as the reference IDE for installation and testing. Lazarus 4.6 was the previous validation target, but it is no longer the active baseline for the 1.1 stabilization line.

Recommended order:

1. Open `packages/LazRibbonRuntime.lpk`.
2. Compile it.
3. Open `packages/LazRibbonDesign.lpk`.
4. Compile and install it.
5. Rebuild the Lazarus IDE when prompted.
6. Restart Lazarus before opening demos.
